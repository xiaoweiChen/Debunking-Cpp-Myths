第7章

—— 若不思重构「物道」与「函术」，则……

在 C++ 中实现并行与并发，过去往往依赖于第三方库（如 Boost）或直接使用操作系统提供的底层原语，这对开发者来说既复杂又容易出错。随着函数式编程特性的引入，例如 lambda 表达式和更强大的标准库支持，在特定约束条件下，并行与并发的使用门槛已显著降低。

本章将重点探讨以下内容：

  • 并行与并发的定义辨析
  • 并行并发编程中的常见陷阱
  • 函数式编程的破局之道
  • 参与者模型(Actor Model)实践
  • 当前尚未实现的能力边界
